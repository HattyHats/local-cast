LOCAL-CAST ARCHITECTURE

Welcome to Local-Cast, a decentralized, browser-based peer-to-peer network designed for secure and ephemeral file sharing and communication.

Core Features

  • Decentralized P2P: All data is transferred directly between browsers via WebRTC. No central server stores your files.
  • Proximity Radar & Comm-Links: Visualizes all connected peers orbiting the Host. The Host can click any blip to open an encrypted 1-on-1 Whisper channel with that guest. While in a Whisper channel, click the green telephone icon to establish a secure, serverless WebRTC Audio Call.
  • Dead Drops (Hidden Files): Right-click any folder to toggle it as a "Dead Drop" (it will turn 50% transparent). Dead drops are completely invisible to guests on the network. To reveal and browse your hidden dead drops, type /deaddrop in the Host search bar. To un-hide a dead drop, simply right-click the transparent folder again and toggle it back to normal.
  • Honey-Pot Traps: Right-click any folder to toggle it as a "Honey-Pot" (it will glow neon pink). When guests attempt to unlock it, the system logs their failed password attempts. Upon 3 failed strikes, their connection is instantly severed and their UI is permanently locked down.
  • P2P Media Streaming: Double-click media files (audio/video/images) to stream them instantly over the P2P connection without having to fully download them first.
  • Burn Notice: The "Destroy Network" button obliterates all active connections, wipes all session data, and permanently shreds the encrypted local storage filesystem, leaving zero trace.

Security & Privacy Transparency

Is the connection secure?
Yes. WebRTC mandates End-to-End Encryption (E2EE) using DTLS and SRTP protocols. No middleman (not even our signaling server) can intercept, read, or listen to your files or audio calls. The browser sandbox also prevents the app from accessing your hard drive or OS.

Can this be used with strangers?
Yes, BUT it is NOT anonymous. Because this is a Peer-to-Peer network, the browsers must exchange IP addresses to connect. Anyone you connect with can see your public IP address using basic networking tools. If you need to stay completely anonymous, all parties must use a VPN before connecting.

To invite guests, they just need to scan your QR code or navigate to the connection URL. Stay secure.
